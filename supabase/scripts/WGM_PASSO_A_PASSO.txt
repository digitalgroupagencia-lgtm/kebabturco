LIGAÇÃO KEBAB ↔ PDV WGM — ORDEM DOS PASSOS
==========================================

PASSO 1 — Supabase do PDV (Flow Operations / WGM)
   Script: WGM_PDV_KEBAB_COMPLETO.sql
   • Troca demo por Kebab Turco
   • Cria chaves de ligação
   • Mostra tabela com IDs das lojas Gandia e Playa
   • Moeda Euro (se a base tiver coluna moeda/currency)

Se o painel ainda mostrar R$ em vez de €:
   Corra no PDV: WGM_PDV_MOEDA_EUR.sql
   Actualize a página (F5)
   Nota: o ecrã «Clientes / Faturamento» ainda pode mostrar R$ — é um bug visual
   desse ecrã no projeto Flow Operations (falta corrigir no Lovable do WGM).

Se aparecerem «Cliente Demo», valores estranhos ou MODO DEMO:
   Corra no PDV: WGM_PDV_LIMPAR_DEMO.sql
   Ou no painel: Configurações → Dados de Teste → Limpar dados de teste
   Pedidos reais só entram quando alguém paga na app Kebab.

Se a chave wg_ NÃO aparecer após o passo 1:
   Corra no PDV: WGM_PDV_CRIAR_API_KEY.sql
   Copie o valor wg_... que aparece nos resultados

PASSO 2 — Lovable do KEBAB (Secrets + Publish)
   WGM_INTEGRATION_API_KEY = chave wg_... (do passo 1 ou CRIAR_API_KEY)
   WGM_INBOUND_WEBHOOK_SECRET = webhook_secret (consulta no PDV abaixo)
   Publish no Lovable (Kebab) — OBRIGATÓRIO para publicar os servidores:
     wgm-sync-dispatch e wgm-inbound-webhook (Cloud → Edge functions)

   Consulta webhook no PDV (SQL Editor):
   SELECT webhook_secret FROM proprioapp_webhook_config c
   JOIN companies co ON co.id = c.company_id
   WHERE co.slug = 'kebab-turco' LIMIT 1;

PASSO 3 — Supabase do KEBAB (não do PDV)
   a) Se ainda não correu: APLICAR_WGM_INTEGRACAO.sql (só uma vez)
   b) Corra KEBAB_ATIVAR_PONTE_WGM.sql — activa a ponte (não dá erro)
   c) Descomente as 2 linhas UPDATE no mesmo script e cole os IDs do passo 1
   d) Corra só essas 2 linhas UPDATE (ou o script inteiro outra vez)

IMPORTANTE:
   • WGM_PDV_KEBAB_COMPLETO.sql → só no PDV (Flow Operations)
   • KEBAB_ATIVAR_PONTE_WGM.sql → só no Kebab (SnapOrder)

PASSO 4 — Admin do Kebab (opcional se o SQL já ligou)
   Menu → Ponte PDV WGM → activar
   Unidades → confirmar ID de cada loja

PASSO 5 — Teste
   Faça um pedido de teste pago na app
   No PDV: Pedidos & Vendas → deve aparecer com origem app

Se o PDV mostrar 0 € / sem pedidos reais:
   A ponte NÃO envia histórico sozinha — só pedidos novos quando ficam pagos.
   No KEBAB: KEBAB_DIAGNOSTICO_PONTE_WGM.sql (ver fila e erros)
   No KEBAB: KEBAB_REENVIAR_PEDIDOS_PDV.sql (reenviar pedidos pagos antigos)
   Admin Kebab → Ponte PDV WGM → Processar fila pendente

Scripts antigos (substituídos pelo completo no PDV):
   WGM_SETUP_NEXUSOPS.sql + WGM_TROCAR_DEMO_POR_KEBAB.sql
   → use só WGM_PDV_KEBAB_COMPLETO.sql no PDV
