KEBAB TURCO — PUBLICAR IPHONE NA NUVEM (sem Xcode no Mac)
============================================================

O Codemagic compila num Mac na nuvem e envia o pacote para a App Store Connect.
Você só configura uma vez no browser.

────────────────────────────────────────────────────────────
PARTE 1 — Criar conta Codemagic (5 min)
────────────────────────────────────────────────────────────

1. Abra https://codemagic.io/signup
2. Entre com a conta GitHub (digitalgroupagencia-lgtm / kebabturco)
3. Autorize o acesso ao repositório kebabturco

────────────────────────────────────────────────────────────
PARTE 2 — Chave Apple no Codemagic (5 min)
────────────────────────────────────────────────────────────

1. Codemagic → ícone equipa → Team settings → Team integrations
2. Developer Portal → Connect (ou Manage keys)
3. Preencha EXACTAMENTE:
   Nome da chave: kebabturco
   Issuer ID: 1c7161d8-4bfa-43b3-9ac0-05624727898a
   Key ID: F7T5LX2NLW
   Ficheiro .p8: AuthKey_F7T5LX2NLW.p8 (Downloads)
4. Save

(O nome "kebabturco" tem de ser igual ao codemagic.yaml)

────────────────────────────────────────────────────────────
PARTE 3 — Certificado de distribuição (5 min)
────────────────────────────────────────────────────────────

1. Team settings → codemagic.yaml settings → Code signing identities
2. Separador iOS certificates → Generate certificate
3. Reference name: kebabturco_appstore
4. Tipo: Apple Distribution
5. Chave API: kebabturco
6. Create certificate

(Se já existir certificado «kebabturco_appstore» na lista, ignore este passo.)

────────────────────────────────────────────────────────────
PARTE 4 — Perfil App Store (OBRIGATÓRIO — falta este passo!)
────────────────────────────────────────────────────────────

O workflow da App Store precisa do perfil kebabturco_appstore.
Só ter kebabturco_dev (Development) NÃO chega.

A) No Codemagic (browser):
   1. Settings → Code signing identities
   2. Separador iOS provisioning profiles
   3. Botão «Fetch profiles»
   4. Em «App Store profiles», marque:
      Kebab Turco App Store TapToPay Final (net.kebabturco.app)
   5. «Download selected»
   6. Reference name: kebabturco_appstore
   7. Confirme que aparece na tabela (tipo app_store, não development)

B) Certificado na mesma página (iOS certificates):
   Deve existir kebabturco_appstore (production) — já tem.

C) Start new build → workflow «Kebab Turco — iPhone App Store»

Se o build disser "No matching profiles found" (método antigo):

A) Criar perfil na Apple (browser):
   1. developer.apple.com → Certificates, Identifiers & Profiles → Profiles
   2. Botão + → Distribution → App Store Connect → Continue
   3. App ID: net.kebabturco.app → Continue
   4. Certificado: Apple Distribution (GROUP EURO BUSINESS) → Continue
   5. Nome: Kebab Turco App Store → Generate

B) Certificado no Codemagic (se ainda não fez upload do .p12):
   1. Code signing → iOS certificates → Upload
   2. Ficheiro .p12 descarregado do Codemagic
   3. Password: (a que guardou ao gerar)
   4. Reference name: kebabturco_appstore

C) Perfil no Codemagic:
   1. iOS provisioning profiles → Fetch profiles
   2. Escolher perfil App Store de net.kebabturco.app
   3. Reference name: kebabturco_appstore
   4. Download selected

D) Start new build outra vez

────────────────────────────────────────────────────────────
PARTE 4b — Development / Tap to Pay no iPhone (configurar UMA vez)
────────────────────────────────────────────────────────────

A) Registar o iPhone na Apple:
   1. developer.apple.com → Dispositivos → +
   2. Nome + UDID do iPhone de teste

B) Certificado Development no Codemagic:
   1. Team settings → Code signing identities → iOS certificates
   2. Generate certificate
   3. Reference name: kebabturco_dev
   4. Tipo: Apple Development (NÃO Distribution)
   5. Chave API: kebabturco
   6. Create certificate
   7. Descarregar .p12 e guardar a password (só uma vez)

C) Perfil Development no Codemagic:
   1. iOS provisioning profiles → Fetch profiles
   2. Em Development profiles, marcar perfil de net.kebabturco.app
      (tem de incluir o iPhone registado + Tap to Pay no App ID)
   3. Reference name: kebabturco_dev
   4. Download selected

   Se não aparecer perfil Development: criar na Apple
   (Perfis → + → Desenvolvimento → iOS App Development → net.kebabturco.app
    → certificado Development → marcar iPhone → Gerar)
   e repetir Fetch profiles.

D) Build no Codemagic:
   Workflow: Kebab Turco — iPhone Development (Tap to Pay)
   (NÃO usar o workflow App Store para testes)

────────────────────────────────────────────────────────────
PARTE 5 — Ligar o repositório (3 min)
────────────────────────────────────────────────────────────

1. Codemagic → Add application → kebabturco (GitHub)
2. Escolha: codemagic.yaml (não Flutter workflow editor)
3. Start your first build → workflow "Kebab Turco — iPhone App Store"

────────────────────────────────────────────────────────────
PARTE 6 — Depois do build OK (automático, SEM Transporter)
────────────────────────────────────────────────────────────

O Codemagic envia o pacote para a Apple sozinho. NÃO precisa de
Transporter nem de Xcode no Mac.

1. Build verde no Codemagic (workflow «Kebab Turco — iPhone App Store»)
2. Esperar 15–45 min
3. appstoreconnect.apple.com → Kebab Turco
   • TestFlight → nova compilação «A processar» → «Pronta para testar»
   • Distribuição → 1.1.1 → Adicionar compilação → escolher a build
4. Adicionar para revisão (loja oficial)

Se aparecer «post-processing failed» em builds antigos: ignore — o .ipa
pode ter chegado na mesma. Nas builds novas isso foi corrigido.

NUNCA é preciso descarregar .ipa nem usar Transporter no Mac.

────────────────────────────────────────────────────────────
NOTAS
────────────────────────────────────────────────────────────

• Plano gratuito Codemagic: ~500 min/mês (1 build iOS ≈ 10–20 min)
• Não precisa de Xcode no seu Mac
• Cada push em main dispara build automático (pode desligar depois)
• Erro de certificado: ver Partes 3 e 4 outra vez
